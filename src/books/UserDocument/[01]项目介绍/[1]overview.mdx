## 这是什么

**WePubReact 是一个 React 工程模板**，为微信公众号等平台提供可定制、统一化的内容编辑脚手架。

要理解这个项目的价值，需要先理解微信公众号的本质。微信公众号的文章内容，本质上就是 HTML 加上 inline style 的 CSS，即一个受限支持的网页。既然是网页，理论上可以用任何前端技术来生成。

市面上的微信编辑器采用了不同的思路。它们提供了大量的预设模板和组件，用户可以在网页中通过拖拽的方式直观地编辑内容，操作简单，上手容易。这种方式的优势在于易用性，但存在一个根本性的限制：用户只能使用编辑器提供的模块和样式。如果预设的模板不符合需求，或者需要实现某种独特的视觉效果，就难以实现。

React 提供了另一种解决方案。通过用代码来描述内容，用户获得了完全的控制权。每一个元素、每一种样式、每一个动画效果，都可以由自己定义。这意味着可以设计出完全符合品牌调性的内容样式，可以构建自己的模块库，可以实现任何能够想到的交互效果。公众号不再受限于编辑器的预设功能，而是由创造力来决定。

这种基于代码的方式在 AI 时代具有更大的优势。编写代码的程序化特性，使得像 Cursor、Claude Code 这样的 AI 工具能够很好地理解和生成代码。用户只需要用自然语言描述想要的效果，AI 就能帮助生成相应的代码。这降低了技术门槛，让非技术背景的内容创作者也能够享受到代码化编辑的灵活性。

因此，WePubReact 的定位非常明确：**它不是为了取代现有的微信编辑器**。那些编辑器在快速排版、简单场景下依然具有价值。WePubReact 的目标是提供脚手架式的模板功能，为那些需要深度定制、追求统一化风格的用户提供基础框架。可以将其视为一个起点，一个让用户能够按照自己的意愿构建公众号内容的基石。

## 核心功能

### PubEditor 发布编辑器

这是整个项目的核心工具，采用经典的三栏布局设计，将工作流程清晰地分为三个区域：左侧是文章选择器，可以快速切换不同的编辑目标；中间是实时预览区域，所见即所得地展示当前效果；右侧是参数调整面板，提供了丰富的配置选项。

这种设计的优势在于工作流的高度集成化。你不需要在多个窗口之间来回切换，也不需要繁琐的保存-刷新-查看的循环。在右侧调整参数后，中间的预览会实时更新，让你能够立即看到改动的效果。当一切都调整满意后，只需一键复制生成的 HTML 代码，就可以直接粘贴到公众号后台。整个过程流畅自然，无需复杂的配置，开箱即用。

编辑器内置了智能的代码生成引擎，能够根据你的参数自动生成优化过的 HTML 和 CSS 代码。这些代码经过精心设计，既保证了在各种平台上的兼容性，又尽可能保持了代码的简洁和可维护性。无论是简单的图文排版，还是复杂的动画效果，都能够通过直观的参数配置来实现。

### SVG 动画组件库

这是多年开发和实践中积累的 SVG 动画组件集合。每一个组件都是经过实际项目验证的，不是简单的演示代码。这些组件涵盖了常见的交互场景：滑动效果可以让元素平滑地进入和退出视野，点击交互提供了丰富的用户反馈机制，翻页动画实现了流畅的页面切换体验，还有各种自定义的过渡特效，可以根据具体需求进行组合和调整。

所有组件都是用 React 实现的，这意味着它们完全遵循 React 的组件化思想。你可以轻松地将这些组件集成到自己的项目中，也可以基于它们进行二次开发。每个组件都提供了清晰的接口和详尽的文档，不需要从零开始编写复杂的动画逻辑，拿来即用，大大提高了开发效率。

组件库的设计理念是"渐进增强"。基础的动画效果实现简单、性能优秀，适合大多数场景。对于有特殊需求的场景，组件提供了丰富的自定义选项，可以实现更复杂的效果。同时，所有组件都经过了性能优化，确保在移动设备上也能流畅运行。

### SVG 转 React 组件

在日常开发中，经常会有将设计师提供的 SVG 文件转换为 React 组件的需求。手动转换不仅耗时，还容易出错。这个工具自动化了这个过程，让你能够专注于更有价值的开发工作。

工具的使用流程非常简单：将 SVG 代码粘贴到输入区域，然后根据需要选择转换选项。你可以选择是否保留原始的 class 属性，这对于保持样式一致性很有帮助。尺寸处理方式也提供了多种选择，可以保持原始尺寸、设置为固定尺寸，或者让尺寸自适应容器。导出格式也很灵活，可以选择函数组件或类组件，是否使用 TypeScript 等等。

转换后的代码格式规范、结构清晰，可以直接集成到项目中。工具还会对 SVG 代码进行优化，移除不必要的属性和注释，确保生成的代码简洁高效。对于需要大量使用 SVG 图标的项目来说，这个工具能够节省大量的时间和精力。

### 颜色工具集

颜色是视觉设计中最重要的元素之一。这个颜色工具集提供了完整的颜色管理功能，涵盖了从颜色选择到格式转换的全部需求。

多格式转换功能支持在 HEX、RGB、HSL、OKLCH 等常见颜色格式之间自由转换。每种格式都有其适用场景：HEX 适合在代码中使用，RGB 适合需要透明度的场景，HSL 适合进行颜色调整，OKLCH 则提供了更感知均匀的色彩空间。工具会自动处理转换过程中的精度问题，确保颜色的一致性。

Google 配色方案参考集成了 Google Material Design 的经典配色方案，为设计提供了专业的色彩搭配参考。Tailwind 色板集成了流行的 Tailwind CSS 色板，方便在项目中使用一致的颜色系统。可视化取色器支持从屏幕上任意位置取色，所见即所得，让颜色选择变得直观简单。

### 全局设置中心

不同的用户有不同的使用习惯和偏好，设置中心提供了丰富的个性化选项。主题切换支持亮色和暗色两种模式，不仅改变了颜色方案，还调整了对比度和阴影效果，确保在任何光照条件下都有良好的视觉体验。

字体设置包括大小和类型的自定义。你可以选择系统内置的字体，也可以使用自定义字体文件，字体大小的调整范围很广，能够满足不同视力需求。侧边栏宽度的调整让界面布局更加灵活，可以根据屏幕大小和个人习惯找到最合适的配置。

这些设置都会被持久化保存，下次打开应用时会自动应用。设置的改变会实时生效，不需要重启应用，让整个体验更加流畅自然。

## 实验性功能：SlideX

**SlideX 是正在开发中的实验性功能**，它代表了项目未来的一个重要方向。

**目标**：使用原生前端技术实现高级 PowerPoint 的演示效果。

传统的演示文稿软件如 PowerPoint、Keynote 等功能强大，但它们生成的文件格式 proprietary，在 Web 环境中使用受限。而现有的在线演示方案往往功能简化，难以达到专业级别的演示效果。SlideX 试图填补这个空白，用纯前端技术实现专业级的演示体验。

这不是简单的幻灯片切换效果，而是要实现真正的演示体验。平滑的过渡动画让页面之间的切换如丝般顺滑，各种视觉特效增强了演示的感染力，可交互的演示体验让听众能够参与到演示过程中，与微信等平台的兼容确保了演示内容可以方便地分享和传播。

目前该功能正在积极推进中，核心的动画引擎已经初具规模，一些基础的过渡效果已经实现。未来的计划包括更多的特效类型、更丰富的交互方式、以及更完善的编辑功能。这是一个长期的探索项目，目标是在 Web 环境中重新定义演示体验。

## 技术栈

项目采用现代化的前端技术栈：React 18 作为 UI 框架，提供了声明式的编程模型和强大的组件化能力；TypeScript 提供了完整的类型安全，在开发阶段就能发现大部分的错误；Vite 作为构建工具，启动速度极快，热更新几乎是瞬时的；Tailwind CSS v4 作为样式系统，提供了实用的工具类和灵活的主题定制能力；shadcn/ui 作为 UI 组件库，提供了大量高质量的组件。

这套技术栈的组合经过精心选择，各个部分都能够很好地配合。React 18 的并发特性配合 Vite 的快速构建，让开发体验达到了新的高度。TypeScript 的类型检查配合 React 的组件模型，让代码的可维护性大大提升。Tailwind CSS 的实用主义哲学配合 shadcn/ui 的组件化思路，让 UI 开发变得高效而愉悦。

这套技术栈不仅适合当前的开发需求，也为未来的扩展留下了充足的空间。无论是要添加新的功能，还是要优化现有的性能，这套技术栈都能够提供良好的支持。
